### Vraag

#### Wat is een goede Front-End Developer?

### Antwoord

Ik vind het antwoord van Egwuenu Gift in het artikel [Wat is een goede frontend developer](https://css-tricks.com/what-makes-a-good-front-end-developer/) goed omschrijven welke skills een frontend developer moet hebben. Zij zegt namelijk *'... has skilled knowledge of HTML, CSS, JS with a vast understanding of User design thinking as they’ll be building web interfaces with accessibility in mind.'*. Daarbij hoort ook nog dat een frontend developer gemotiveerd moet zijn om nieuwe ontwikkelingen te leren om zijn skills te verbreden.

Zelf ben ik meer een JavaScript Engineer (op basis van [dit artikel](https://css-tricks.com/the-great-divide/)), die ook een goede kennis heeft van HTML, CSS & UX. Omdat ik mij momenteel meer focus op de JavaScript kant van de frontend developer beschouw ik mij daarom meer als JavaScript Engineer dan een UX Engineer.

## Weekly Nerd #2

### Aantekeningen

Zie de aantekeningen van de gastsprekers [hier](https://github.com/tsjuusmei/weekly-nerd-2021/wiki/Aantekeningen-week-2).

### Vraag

#### Ook de tech-industrie bestaat voor een groot deel uit dezelfde type personen en testen vervolgens hun websites bij weer dezelfde types, met stereotypering, vooroordelen en biased uitkomsten tot gevolg. Herken je dit als probleem?

### Antwoord

In het bijbehorende artikel van Adam Fard worden voorbeelden gegeven van verschillende keren waar er geen tot weinig rekening is gehouden met diversiteit van de testpersoon, en dat er zelfs racisme in de alrogitmes zitten van websites. Dit klinkt allemaal best heftig, maar in werkelijkheid denk ik zelf niet dat dit door de algoritmes zelf komt, maar door de makers er van. Ook wordt er gezegd dat het ontwerp van de Fitbit en Apple Watch racistisch zijn doordat ze niet werken op alle huidskleuren, omdat deze waarschijnlijk niet getest zouden worden op mensen met een donkere huid. Ik denk zelf dat dit niet helemaal het geval is, maar dat er een aantoonbare reden voor is. Wel denk ik dat het cruciaal is dat er zo veel mogelijk rekening gehouden moet worden met diversiteit. Iedereen heeft zijn eigen inzichten en ideeën, en wanneer er niet getest is worden er aannames genomen voor bepaalde keuzes. Dit is een groot probleem in mijn ogen, omdat vaak aannames niet kloppen. Dit heb ik zelf meegemaakt met het vak 'Humand centered Design', waar wij gingen ontwerpen voor iemand met een fysieke beperking.

## Weekly Nerd #3

### Vraag

Wat heb ik geleerd in de minor?

### Antwoord

Deze hele minor heb ik veel geleerd op het gebied van Frontend Development. Elk vak gingen we iets dieper in de Development wereld duiken, wat zorgde voor een mooie chronologische kennisverbreding.

#### Web App From Scratch

Hier hebben we een webapplicatie gemaakt in combinatie met een externe API, die client-side wordt gerenderd. Hier had ik al enige ervaring mee, dus heb ik meer geoefend met een eigen API maken, maar ook met een nieuwe API werken en daar de functionaliteiten van uitvinden. Daarnaast heb ik verder geleerd hoe ik het beste `async` en `await` kan gebruiken binnen mijn code, en hoe de opgehaalde data wordt omgezet naar HTML. Ook hebben we gebruik gemaakt van een `router` die de routes regelde, wat het maken met de data een stuk makkelijker maakte.

#### CSS To The Rescue

Bij dit vak ben ik veel wijzer geworden van wat er mogelijk is met CSS. Ook hebben we met restricties en extra uitdagingen gewerkt. Zo mocht ik maar twee kleuren gebruiken, en moest ik ook een functionaliteit maken voor mensen die `reduced motion` aan hebben staan op hun device. Door de twee kleuren restrictie kan je gebruik gaan maken van verschillende functies waardoor bijvoorbeeld de saturatie wordt veranderd, en zo alsnog een andere kleur krijgt te zien dan de ingevoerde kleur. Bij de `reduced` motion heb ik de animaties een heel stuk verminderd, waardoor de look&feel van de website veranderd, maar nogsteeds aanwezig is.

#### Progressive Web App

Hier hebben we de webapplicatie van Web App From Scratch omgebouwd naar een server side gerenderde app. Ook hier had ik al enige ervaring mee, dus ik kon snel aan de slag met Node en Express. Na het omzetten van de app heb ik gefocust op de lighthouse reports van de applicatie, en heb deze geoptimaliseerd.

#### Browser Technologies

Dit vak was erg technisch voor mijn gevoel. Eerst moesten we een opdracht kiezen, en deze progressively enhanced gaan bouwen. Hiervoor heb ik ook goed uitgedacht en uitgetekend wat er allemaal met HTML & CSS mogelijk was, en wat er vervolgens nog voor extra user experience progressively enhanced toegevoegd kon worden. Ik was erg lang bezig met alles werkend te krijgen op technisch vlak, maar heb hier uiteindelijk een mooie en werkende oplevering gemaakt.

#### Real-Time Web

Bij dit vak gingen we aan de slag met `sockets`, waarmee je live met andere gebruikers iets mee kan maken, zoals een chat. Hier heb ik een multiplayer canvas gemaakt, inclusief chat en afbeelding die de gebruikers na kunnen tekenen. Hier heb ik geleerd om met `socket.io` te werken, en met al zijn functies daarbinnen. Dit was ook echt een case van de documentatie lezen en voorbeelden op het internet zoeken, wat heel leerzaam was.

#### Human Centered Design

Dit vak was meer gericht op design, en het testen van en voor iemand met een (in mijn geval) fysieke beperking. Dit was heel erg interessant, omdat je aannames gaat doen en die vervolgens totaal niet blijken te kloppen. Door goede tests af te leggen en te itereren op het ontwerp, kwam je uiteindelijk met een mooie oplossing.
